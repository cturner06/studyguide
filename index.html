Domain 1 of 4
🚀 Deploy Windows Client
How devices get set up, enrolled, and managed for the first time
25–30% of exam
📦 1.1 — Windows Autopilot▶
🔑 New device / shipped from vendor / zero-touch / no IT involvement / out of box / OOBE / remote worker
Windows Autopilot
Device ships from manufacturer to user. Intune takes over at sign-in. Hardware Hash (CSV) must be uploaded to Intune BEFORE shipping. Deployment Profile must be assigned to a device group.
💡 New device + no IT touch = Autopilot
🔑 What must be uploaded BEFORE Autopilot works / device fingerprint / CSV file
Hardware Hash
Unique fingerprint of the device. Without it, Intune does not recognize the device as belonging to your company.
🔑 Kiosk / no user login at boot / device configures itself / digital signage / TPM authenticates
Autopilot Self-Deploying Mode
Device uses its TPM 2.0 chip to authenticate itself to Intune. Zero user interaction required.
🚫 Self-Deploying = DEVICE deploys itself — NOT the user deploying it.
🔑 IT pre-loads apps before shipping / White Glove / ready when box opens
Autopilot Pre-Provisioning — White Glove
IT connects device first, Intune installs all heavy apps, then ships. User just signs in and everything is ready.
🔑 Reset device / repurpose / re-assign to new employee / keep in Intune
Autopilot Reset
Wipes device but keeps it enrolled in Intune and Autopilot-registered. Returns to setup screen.
🔑 Block device use until all policies and apps apply / setup progress screen
Enrollment Status Page — ESP
Screen during Autopilot setup. Can block the user from accessing the desktop until setup is 100% complete.
🔗 1.2 — Co-Management (SCCM + Intune)▶
🔑 SCCM + Intune / Configuration Manager / MECM / migrating to cloud / both systems at once
Co-Management
Lets SCCM and Intune manage the same device simultaneously. Workload sliders decide who manages each feature. Device MUST be Hybrid Azure AD Joined.
💡 SCCM + Intune together = Co-Management
🔑 Who manages compliance / who manages updates / move features from SCCM to Intune
Workload Sliders
Move each feature area independently — Compliance, Updates, Apps, Endpoint Protection. Don't have to move everything at once.
🔑 Test Intune workload on small group first / pilot before full rollout
Pilot Collection
Small test group. Test the slider change here before applying to all devices.
🔑 Connect SCCM to cloud for visibility only / lighter than co-management
Cloud Attach / Tenant Attach
Connects SCCM to Intune for visibility without full workload migration. A lighter starting point.
🔗 1.3 — Azure AD Join Types▶
🔑 Cloud only / no on-premises AD / managed purely by Intune / new or remote worker device
Azure AD Joined
Device exists only in Azure AD. No on-prem domain join. Pure Intune management.
🔑 On-prem AD + Azure AD on same device / existing domain-joined device needs Intune / required for co-management
Hybrid Azure AD Joined
ONE device in both on-premises AD and Azure AD simultaneously.
🚫 Hybrid = one device in two places. NOT two separate devices.
🔑 Personal BYOD device / accessing company apps / not fully managed by IT
Azure AD Registered
Device just registered — no MDM policy on the device. App Protection Policies (MAM) protect the corporate app data.
🖥️ 1.4 — Other Deployment Methods▶
🔑 Custom gold image / on-premises imaging / IT builds image / MDT
MDT — Microsoft Deployment Toolkit
On-premises imaging. IT builds a custom Windows image and deploys it. Requires IT to touch each device or use PXE boot.
🔑 Upgrade Windows 10 to 11 / keep apps and data / no wipe / same device
In-Place Upgrade
Upgrades OS without wiping. User's apps, settings, and data are preserved.
Domain 2 of 4
🔐 Manage Identity & Compliance
Who can access what, under what conditions, and how access is enforced
15–20% of exam
🚪 2.1 — Conditional Access — The Enforcer▶
🔑 Block access / allow only if compliant / restrict login / require MFA / untrusted location / Zero Trust / IF-THEN rule
Conditional Access
The bouncer. Checks WHO (user) + WHAT device (compliance) + FROM WHERE (location) + INTO WHAT APP. Then: Allow, Block, or Require MFA. Always needs a Compliance Policy as its partner.
⚡ COMBO: "Allow access ONLY if device is secure" = Compliance Policy (grades) + Conditional Access (enforces). Both required.
💡 ANY access restriction = Conditional Access
🔑 Trusted IP range / office network / outside triggers extra check
Named Locations in Conditional Access
Define trusted IP ranges. Outside = risky = may trigger MFA or block.
🔑 Suspicious sign-in / credential breach / impossible travel / risky user
Entra ID Protection → feeds Conditional Access
Microsoft AI flags unusual sign-ins. Generates risk signals CA policies act on — block, require MFA, force password reset.
📋 2.2 — Compliance Policies — The Rulebook▶
🔑 Device must be secure / healthy / BitLocker on / minimum OS / no jailbreak / firewall / antivirus running
Compliance Policy
You write the rules defining a "safe" device. Intune evaluates and stamps Compliant ✅ or Not Compliant ❌. Separate policies for Windows, iOS, Android, macOS.
🚫 Compliance Policy ONLY evaluates. It does NOT block access. Conditional Access does the blocking.
💡 Rulebook that grades the device = Compliance Policy
🔑 Give users time to fix device before being blocked / days before marking non-compliant
Grace Period (inside Compliance Policy)
Set X days before a device is officially marked non-compliant. Prevents locking users out immediately when a new policy deploys.
👤 2.3 — Authentication & Identity▶
🔑 Face recognition / fingerprint / biometrics / PIN login / passwordless
Windows Hello for Business
Replaces passwords with biometrics or PIN. Credential stays on device — never travels over the network. Deployed via Intune Configuration Profile.
🚫 Windows Hello REPLACES passwords. It is not a password manager.
🔑 Two-factor / password + phone / something you know + something you have
MFA — Multi-Factor Authentication
Enforced through Conditional Access policies — not per-user settings.
🔑 User forgot password / resets own password / reduce help desk calls
SSPR — Self-Service Password Reset
Users reset using email, phone, or authenticator. No IT ticket. Configured in Entra ID.
🔑 2.4 — Roles & Privileged Access▶
🔑 Who can manage Intune / assign admin permissions / least privilege for IT staff
Azure AD Roles — RBAC
Global Admin (everything), Intune Admin (devices), Security Admin (security policies), Help Desk Operator (basic support). Always assign minimum permissions needed.
🔑 Temporary admin access / just-in-time / time-limited / no permanent admin rights
PIM — Privileged Identity Management
Admin activates role for a limited window with justification. Access expires automatically. Reduces always-on admin attack surface.
💡 Temporary / time-limited admin = PIM
Domain 3 of 4 — LARGEST DOMAIN
🛡️ Manage, Maintain & Protect Devices
Configuring, updating, securing, and acting on enrolled devices
40–45% of exam — Study this first
⚙️ 3.1 — Configuration Profiles▶
🔑 Push any setting to device / WiFi / VPN / email / device restrictions / OMA-URI / kiosk / configure remotely
Configuration Profile
Remote control for device settings. Types: WiFi, VPN, Email, Device Restrictions, Kiosk, Certificate, Admin Templates. Assigned to User groups or Device groups.
💡 Any device setting pushed remotely = Configuration Profile
🔑 Setting NOT in the Intune GUI / custom setting / need an exact URI path
OMA-URI Custom Profile
Enter the OMA-URI path from Microsoft docs in a Custom profile type. Advanced option for edge cases.
🔑 Same as Group Policy but through Intune cloud / replicate on-prem GPO
Administrative Templates Profile
Mirrors Group Policy settings from Intune. Used when migrating from on-prem GPO to cloud management.
🔑 Two profiles conflicting / same setting assigned twice with different values
Conflict — Most Restrictive Wins
Shows as "conflict" in device detail. Most restrictive value applies.
🔄 3.2 — Windows Update for Business▶
🔑 Schedule patches / stagger rollout / pilot group first / defer updates / control timing
Update Rings
Create rings with different deferral periods. Quality updates defer up to 30 days. Active Hours prevents restarts during work hours.
💡 Control update timing = Update Rings
🔑 Bad update causing problems / emergency stop / halt patches
Pause Update Ring
Emergency brake. Maximum 35 days. Updates resume automatically after pause expires.
🔑 Critical security patch ASAP / force update now / ignore deferral / zero-day
Expedite Policy
Forces a specific quality update immediately on all targeted devices, bypassing all deferral settings.
🔑 Control which Windows version / major OS upgrade / Windows 10 to 11
Feature Update Policy
🚫 Update Rings = monthly patches. Feature Update Policy = major OS version. Completely different things.
🔑 Slow network / high bandwidth / devices sharing update files / save internet usage
Delivery Optimization
Devices share update files with each other on local network. Reduces internet bandwidth.
💡 Slow network + update bandwidth = Delivery Optimization
🔑 Monitor update health / which devices got the patch / update status dashboard
WUfB Reports — Update Compliance
Dashboard showing update status per device — received, pending, failed.
🛡️ 3.3 — Microsoft Defender▶
🔑 Configure antivirus / real-time protection / scan schedule / exclusions across all devices
Intune Endpoint Security → Antivirus Policy
Configure Defender Antivirus via Intune — not Group Policy, not local settings.
🔑 Block Office macros / block scripts from email / block risky app behaviors
ASR Rules — Attack Surface Reduction
Preventive rules blocking high-risk behaviors. Audit, Warn, or Block mode. Via Intune Endpoint Security policy.
🔑 Advanced threats / beyond antivirus / behavioral detection / threat hunting
EDR — Endpoint Detection and Response
Catches sophisticated attacks antivirus misses. Requires Defender for Endpoint Plan 2.
🔑 Ransomware spreading / active infection / cut from network immediately
Device Isolation — Defender for Endpoint
Cuts all network access while keeping Defender connection alive. Stops spread in seconds.
🚫 ASR is preventive. Isolation is for active infections already happening.
🔒 3.4 — BitLocker & Encryption▶
🔑 Encrypt hard drive / lost laptop / data at rest / TPM chip / stolen device
BitLocker
Full disk encryption. Stolen drive = data completely unreadable. Managed via Intune Endpoint Security → Disk Encryption. Requires TPM chip.
💡 Drive encryption + lost device = BitLocker
🔑 Where is the recovery key / user locked out of encrypted drive / IT needs the 48-digit key
Entra ID — from the Intune device page
Recovery key automatically escrowed when BitLocker is managed by Intune.
🔑 Mac disk encryption / encrypt MacBook / FileVault managed by Intune
FileVault via Intune Disk Encryption Policy
Apple's BitLocker equivalent. Managed and escrowed through Intune the same way.
📡 3.5 — Remote Device Actions▶
🔑 Stolen corporate device / remove everything / factory reset remotely
Full Wipe
Factory reset. Removes ALL data — company and personal.
🚫 NEVER Full Wipe a personal/BYOD device.
🔑 BYOD employee leaves / remove company data only / keep personal photos and apps
Retire
Removes company apps, email, policies. Personal data completely untouched. Device un-enrolls.
🔑 Remove only corporate app data / device never enrolled in MDM / MAM-only
Selective Wipe — MAM Wipe
Deletes corporate data inside managed app only. App stays. Device untouched. No MDM enrollment needed.
🔑 Policy not applying / force device to check in now / settings not showing up
Sync Action
Forces device to check in with Intune immediately. First troubleshooting step when a policy isn't applying.
🔑 Device possibly lost / lock screen now / user must enter PIN
Remote Lock
Immediately locks the screen. User must enter PIN or password to unlock.
📧 3.6 — Data Protection▶
🔑 Protect email / prevent forwarding / classify files / sensitivity labels / encrypt documents / Do Not Forward
Microsoft Information Protection — MIP / Purview
Sensitivity Labels (Confidential, Internal, Public) and Rights Management. Works across Office 365, email, Teams, SharePoint. Auto-labeling classifies based on content automatically.
💡 Protect email or files from sharing = MIP
🔑 Protect company data on Windows device locally / separate work from personal on PC
Windows Information Protection — WIP
OS-level data protection on Windows. Note: being deprecated — Microsoft moving toward App Protection Policies instead.
Domain 4 of 4
📦 Manage Applications
Deploying, protecting, and managing apps on enrolled and personal devices
10–15% of exam
📲 4.1 — App Deployment Types▶
🔑 Force install silently / user has no choice / auto-deploy / appears without user doing anything
Required App Assignment
Installs automatically in the background. No user prompt.
💡 Silent automatic install = Required
🔑 User chooses to install / optional app / Company Portal / self-service
Available App Assignment
App appears in Company Portal. User clicks Install if they want it.
🔑 Traditional .exe or .msi / custom desktop application / Win32
Win32 App
Package with IntuneWinAppUtil.exe. Define install/uninstall commands and detection rules.
🔑 Company's own custom-built app / internal proprietary app / LOB
LOB App — Line of Business
Your organization's own app. Upload .msi or .intunewin directly to Intune.
🔑 App shows install pending but never completes / deployment stuck
Check Detection Rules
Detection rules tell Intune the install succeeded. Wrong rules = Intune thinks it failed forever.
🔐 4.2 — App Protection Policies (MAM)▶
🔑 Personal phone / BYOD / copy paste / protect app data / WITHOUT enrolling device / data leakage from apps
App Protection Policy — MAM
Invisible wall around the corporate app. No MDM enrollment required. Works on Outlook, Teams, Edge, OneDrive.
🚫 "Without enrolling the device" = App Protection Policy. Every time. No exceptions.
💡 BYOD + protect only the app = App Protection Policy
🔑 Block copying corporate email to personal Notes / prevent data leaving corporate apps
App Protection Policy — Restrict Copy/Paste
Blocks cut/copy/paste from managed to unmanaged apps. Also blocks Save As to personal storage.
🔑 Employee leaves / remove corporate data from personal phone / leave personal stuff
Selective Wipe — MAM Wipe
Removes corporate data inside managed apps only. App and device completely untouched.
🏢 4.3 — Microsoft 365 Apps▶
🔑 Latest Office features fastest / monthly updates / current channel
Current Channel — Monthly
🔑 Stable / regulated environment / less frequent Office updates / every 6 months
Semi-Annual Enterprise Channel
Best for hospitals, government, regulated industries needing maximum stability.

🚀 Domain 1 — Deploy Windows▶
New device / setup / Autopilot / deployment ▶
→ Windows Autopilot
  • No user login / kiosk → Self-Deploying Mode
  • IT loads apps before shipping → Pre-Provisioning / White Glove
  • Repurpose device to new employee → Autopilot Reset
  • Device not getting profile → Check hardware hash uploaded AND profile assigned to device group
  • Block device until setup complete → Enrollment Status Page (ESP)
SCCM + Intune / migrating / co-management ▶
→ Co-Management

  • Move features from SCCM to Intune → Workload Sliders
  • Test on small group first → Pilot Collection
  • Connect SCCM to cloud for visibility only → Cloud Attach
Which Azure AD join type? ▶
→ Match the scenario
  • Cloud only / Intune only / no on-prem AD → Azure AD Joined
  • On-prem AD + Azure AD / co-management needed → Hybrid Azure AD Joined
  • Personal BYOD device / not fully managed → Azure AD Registered + MAM
🔐 Domain 2 — Identity & Compliance▶

→ Conditional Access

→ Compliance Policy
⚠️ Compliance Policy grades the device. It does NOT block access. Conditional Access blocks.
  • BitLocker required to access SharePoint → Compliance Policy + Conditional Access

  • Credential stays on device — never transmitted over network

→ Match the scenario

  • Suspicious sign-in / credential breach → Entra ID Protection

  • Critical patch bypass deferral → Expedite Policy

  • Slow network / save bandwidth → Delivery Optimization
  • See which devices got the patch → WUfB Reports
Threat / malware / infection / attack ▶
→ Microsoft Defender for Endpoint
  • Block risky behaviors before attack → ASR Rules
  • Active infection / ransomware spreading now → Device Isolation
  • Advanced threat bypassed antivirus → EDR
  • Configure antivirus across all devices → Intune Endpoint Security → Antivirus Policy
Encrypt drive / lost laptop / recovery key ▶
→ BitLocker
  • Where is recovery key? → Entra ID — from Intune device page
  • Mac version? → FileVault via Intune Disk Encryption Policy
  • Hardware requirement? → TPM chip
Remote actions / wipe / remove / sync ▶
→ Pick the right action
  • Stolen corporate device → remove everything → Full Wipe

  • Remove only app data / not MDM enrolled → Selective Wipe (MAM)
  • Policy not applying → force check-in now → Sync Action
